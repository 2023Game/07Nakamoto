1. 先頭につける場合：データが変更できなくなる

構文：const データ型 *変数名
または
構文：データ型 const *変数名

例：const int *p;

ポインタが指しているデータ(値)が変更できなくなる
ポインタの指す先にある中身が変更できない

例：*p = 100;→コンパイルエラー(データ変更不可)

ポインタ自身(アドレスは変更できる
例：p = &another_variable; →OK(ポインタの指し先変更可能)


2. 後ろにつける場合：ポインタのアドレスが変更できなくなる

構文：データ型 *const 変数名;

例：int *const p;

ポインタ自身(ポインタが格納するアドレス)が変更できなくなる。
宣言時に必ず初期化が必要。
ポインタ自身が格納するアドレスを変更できない

例：p = &another_variable;→コンパイルエラー (ポインタの指し先変更不可)

ポインタが指す先のデータ(中身)は変更できる

例：*p = 100;→OK(データ変更可)


3. 両方につける場合：データもポインタも変更できなくなる

構文：const データ型 *const 変数名;

例：const int *const p;

ポインタが指すデータ(中身)と、ポインタ自身のアドレスの両方が変更できない。
全て変更不可能

例：*p = 100;→コンパイルエラー(データ変更不可)
例：p = &another_variable;→コンパイルエラー(ポインタの指し先変更不可)
